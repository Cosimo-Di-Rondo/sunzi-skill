# 工作流组合

> 「多算胜，少算不胜，而况于无算乎！」
> ——《孙子兵法·计篇》
方法论的威力不在于单独使用某一计，而在于**在正确的时机以正确的顺序组合使用**。本 skill 提供三条标准化工作流，定义 skill 调用顺序、步骤间数据传递格式与终止条件。
---

## Workflow 1：新项目评估与启动
**适用场景：** 从零开始新项目/新任务，需先系统评估形势、权衡利弊后再启动执行。  
**触发信号：** 新项目、新任务、从零启动、需先评估再动手、资源与风险需一并看清
### 工作流
```
know-both-sides → strategic-calculation → void-solid-analysis → win-before-fighting
    知彼知己        →      庙算          →     虚实分析        →     先胜后战
```
### 步骤详解
**Step 1：know-both-sides（知彼知己）**
- 目标：摸清己方与对手/环境的关键信息，避免一厢情愿
- 输出格式（传递给 Step 2）：
  ```
  知彼知己摘要：
  - 己方：优势/劣势/约束/可用资源：……
  - 彼方（或环境/市场/竞品）：关键动向与能力边界：……
  - 信息缺口与需补证项：……
  ```
- 终止条件：能回答「我与彼/环境各自关键变量是什么」，且缺口已登记

**Step 2：strategic-calculation（庙算）**
- 输入：Step 1 的知彼知己摘要｜目标：在行动前完成多方案比较与胜算评估，选定战略取向
- 输出格式（传递给 Step 3）：
  ```
  庙算结论：
  - 备选方案：[A]/[B]/…… 及各自胜算要点
  - 选定方案：…… 理由：……
  - 不可接受的风险底线：……
  ```
- 终止条件：有明确选定方案与可接受边界

**Step 3：void-solid-analysis（虚实分析）**
- 输入：Step 2 的庙算结论｜目标：在选定方向上识别何处为实、何处为虚，何处可击、何处需避
- 输出格式（传递给 Step 4）：
  ```
  虚实研判：
  - 我方应守之实（必须保住的能力/阵地）：……
  - 可击之虚（对方或环境中的薄弱点）：……
  - 我方需补之虚（当前短板）：……
  ```
- 终止条件：虚实清单清晰，且与选定方案一致

**Step 4：win-before-fighting（先胜后战）**
- 输入：Step 3 的虚实研判｜目标：在动手前把胜算条件设计到位，明确「胜兵先胜而后求战」的启动条件
- 输出格式：
  ```
  先胜条件与启动包：
  - 启动前必须满足的胜算条件（可检查项）：……
  - 不满足时：暂缓/调整策略：……
  - 正式启动时的第一步行动与成功判据：……
  ```
- 终止条件：有可执行的启动判据与第一步；或结论为「条件未备，暂缓」并给出补条件路径

## Workflow 2：技术难题攻坚
**适用场景：** 复杂技术问题，根因或最优路径不明，需系统找到突破点并收敛为可执行解法。  
**触发信号：** 疑难 bug、性能/架构瓶颈、多因素耦合、反复无果、多路线抉择
### 工作流
```
know-both-sides → void-solid-analysis → orthodox-unorthodox → momentum-timing → swift-resolution
    知彼知己        →     虚实分析        →      奇正           →    势与机        →     速决
```
### 步骤详解
**Step 1：know-both-sides（知彼知己）**
- 目标：对齐现象、复现条件、已排除项、约束与技术边界
- 输出格式（传递给 Step 2）：
  ```
  技术情境摘要：
  - 现象与影响范围：……
  - 复现/触发条件；已验证排除：……
  - 约束（时间/兼容/安全）：……
  ```
- 终止条件：问题可被第三方复述而不歧义

**Step 2：void-solid-analysis（虚实分析）**
- 输入：Step 1 的技术情境摘要｜目标：区分症状与根因、主路径与干扰项，标出薄弱「虚」
- 输出格式（传递给 Step 3）：
  ```
  虚实剖分：
  - 表面现象 vs 可疑根因：……
  - 主攻链路（最值得验证的一条）：……
  - 次要干扰项（先搁置）：……
  ```
- 终止条件：有一条可验证的主攻假说

**Step 3：orthodox-unorthodox（奇正）**
- 输入：Step 2 的虚实剖分｜目标：正兵（基线验证）与奇兵（非常规探测）配合，避免单路径钻牛角尖
- 输出格式（传递给 Step 4）：
  ```
  奇正组合：
  - 正兵（基线验证步骤）：……
  - 奇兵（探索性步骤）：……
  - 二者如何互相印证/排除：……
  ```
- 终止条件：正奇步骤可执行且能判定假说方向

**Step 4：momentum-timing（势与机）**
- 输入：Step 3 的奇正组合｜目标：排定验证顺序与时机（何时加压、何时收束），形成「势」
- 输出格式（传递给 Step 5）：
  ```
  势与机：
  - 推荐执行顺序与理由：……
  - 关键决策点（何时转攻/转守）：……
  - 资源与时间窗口：……
  ```
- 终止条件：有明确顺序与决策点

**Step 5：swift-resolution（速决）**
- 输入：Step 4 的势与机｜目标：在验证路径上快速收敛，产出可落地的修复或架构结论
- 输出格式：
  ```
  速决结论：
  - 采纳方案：……
  - 残留风险：……
  - 若假说被证伪：返回 Step 2 时需携带的结构化输入：……
  ```
- 终止条件：技术问题被解决，或明确「需换主攻链路」并带着结构化输入返回 Step 2

## Workflow 3：应变与迭代
**适用场景：** 计划遇阻或环境/需求变化，需在变动中重划虚实、迂回调度，多轮直至稳定。  
**触发信号：** 计划中断、依赖变更、需求反转、上线风险、需多轮调整
### 工作流
```
adapt-to-win → void-solid-analysis → indirect-approach → swift-resolution → adapt-to-win（循环）
  因敌制胜      →     虚实分析        →      迂直           →     速决        →  因敌制胜（下一轮）
```
### 步骤详解
**Step 1：adapt-to-win（因敌制胜）**
- 目标：识别变化源与新约束，重新定义「赢」在当前阶段的含义
- 输出格式（传递给 Step 2）：
  ```
  应变输入：
  - 变化/阻碍是什么：……
  - 新目标与非目标：……
  - 不可逾越的红线：……
  ```
- 终止条件：新情境与目标表述一致，干系人可共识

**Step 2：void-solid-analysis（虚实分析）**
- 输入：Step 1 的应变输入｜目标：在新情境下重划虚实，找出当前最致命的短板与可乘之机
- 输出格式（传递给 Step 3）：
  ```
  新虚实图：
  - 必须立刻补的实：……
  - 可借的虚：……
  ```
- 终止条件：与 Step 1 的目标对齐

**Step 3：indirect-approach（迂直）**
- 输入：Step 2 的新虚实图｜目标：以迂为直，设计非常规或分步路径绕开硬碰硬
- 输出格式（传递给 Step 4）：
  ```
  迂直方案：
  - 间接路径（步骤）：……
  - 与「直取」相比的收益/代价：……
  ```
- 终止条件：有可执行路径

**Step 4：swift-resolution（速决）**
- 输入：Step 3 的迂直方案｜目标：快速落地本轮调整并观察效果
- 输出格式（传递给 Step 5 / 下一轮 Step 1）：
  ```
  本轮结果：
  - 已执行：……
  - 效果与数据：……
  - 新问题/残留风险：……
  ```
- 终止条件：本轮行动已闭环复盘

**Step 5：adapt-to-win（下一轮循环入口）**
- 输入：Step 4 的本轮结果｜目标：判断是否需继续应变；若需，带着新情境回到 Step 1
**循环终止条件：** 满足下列之一即可退出：（1）新情境下目标已达成且风险可接受；（2）多源信息显示无新的重要变化且干系人确认；（3）继续迭代收益低于成本，明确冻结方案或改走 Workflow 1/2。

## 工作流选择指南

| 你的情况 | 推荐工作流 |
|---------|-----------|
| 从零开始新项目/新任务，需系统评估后再启动 | Workflow 1：新项目评估与启动 |
| 复杂技术难题，需最佳突破路径并收敛解法 | Workflow 2：技术难题攻坚 |
| 计划遇阻或环境变化，需多轮应变与迭代 | Workflow 3：应变与迭代 |
| 以上都不完全符合 | 回到 `strategic-thinking` 或单 skill 调度 |
## 数据传递规则
工作流中每个步骤的**输出必须显式传递给下一步骤**。具体做法：① 每步结束时写出格式化「传递给下一步的信息」；② 下步开始时确认并引用上一步输出；③ 不得跳步（除非工作流中相邻两步直接相连）。
## 工作流中断处理
**信息不足** → 插入 `know-both-sides`（或等价补充调查）后从当前步继续。**主攻假说被证伪** → Workflow 2 回 `void-solid-analysis`，Workflow 1 可回 `strategic-calculation` 或 `know-both-sides`。**环境/目标剧变** → Workflow 3 从 `adapt-to-win` 重入环；超出本轮假设时先用 Workflow 1 再评估。**需全局重排** → 暂停并输出阶段结论，改 Workflow 1 或单独 `strategic-calculation` 再规划。
