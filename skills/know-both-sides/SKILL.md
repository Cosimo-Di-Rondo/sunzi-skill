# 知彼知己

> "知彼知己者，百战不殆；不知彼而知己，一胜一负；不知彼不知己，每战必殆。"
> —— 孙子《孙子兵法·谋攻篇》

> "明君贤将，所以动而胜人，成功出于众者，先知也。先知者，不可取于鬼神，不可象于事，不可验于度，必取于人，知敌之情者也。"
> —— 孙子《孙子兵法·用间篇》

## 核心原则

**胜利来自对双方情况的透彻了解。既了解对手又了解自己，百战不会有危险；只了解自己不了解对手，胜负各半；既不了解对手也不了解自己，每战必败。**

"先知"——预先掌握情报——是一切正确决策的基础。先知不能靠鬼神、不能靠类比推测、不能靠主观臆度，只能靠实际调查来获取。情报工作不是可选项，而是行动的前提。

## 不适用场景

**以下情况不需要调用此 skill：**
- 上下文已经完整清晰，用户已提供了充分的信息
- 琐碎任务，不涉及决策和风险判断
- 用户已经做过充分的调研并明确告知了结论
- 纯粹的执行阶段——情报已收集完毕，当前任务是动手做

## 何时使用

你应该在以下情况调用此 skill：

- 进入一个不熟悉的代码库、系统或领域
- 要做决策但信息不足，存在明显的知识盲区
- 需要理解需求再设计方案——先搞清楚"敌情"
- 调试一个复杂 bug——需要同时理解 bug 的表现（彼）和系统的机制（己）
- 做竞品分析或技术对比
- 感觉自己在凭假设行动，而非基于事实
- 庙算决策中发现信息不足，需要补充情报

## 方法流程

### 第一步：知彼——深入调查对手/问题

> "知彼知己者，百战不殆。"——《谋攻篇》

系统调查目标对象（问题、系统、竞争对手、需求方）：
- 它的核心逻辑和运作机制是什么？
- 它的强项在哪里？弱项在哪里？
- 它最近发生了什么变化？趋势如何？
- 有哪些约束条件和隐含规则？

**调查方法**：直接阅读源码、运行测试、查看日志、分析数据——"必取于人，知敌之情者也"，不能靠猜测。

### 第二步：知己——诚实评估自身

> "不知彼而知己，一胜一负。"——《谋攻篇》

对自身做诚实的审视：
- 我/团队具备什么能力？缺少什么能力？
- 可用资源（时间、人力、工具）有多少？
- 有什么硬约束（截止日期、兼容性要求、技术限制）？
- 过往类似经历中的教训是什么？

**警惕**：知己比知彼更难——人容易高估自己的能力、低估自己的约束。

### 第三步：交叉对比——寻找优势交汇点

将"彼"和"己"的信息放在一起比较：
- 我的强项与对手/问题的弱项在哪里重合？——这是最佳攻击点
- 我的弱项与对手/问题的强项在哪里重合？——这是最大风险点
- 哪些地方势均力敌？——这些地方需要额外策略

### 第四步：识别盲区——还有什么不知道

> "先知者，不可取于鬼神，不可象于事，不可验于度。"——《用间篇》

诚实列出仍然未知的信息：
- 有哪些关键信息我还没有获取到？
- 有哪些假设我还没有验证？
- 有哪些领域我完全没有涉及？
- 这些盲区可能带来什么风险？

**原则**：宁可承认"不知道"，也不能用猜测填补盲区。

### 第五步：风险评估——不知的代价

> "不知彼不知己，每战必殆。"——《谋攻篇》

评估盲区带来的风险等级：
- 若某个盲区被忽视，最坏后果是什么？
- 这个风险是否可承受？
- 是否需要在行动前填补这个盲区？
- 填补盲区的成本与不填补的风险，哪个更大？

### 第六步：综合为可执行情报

> "故用间有五：因间、内间、反间、死间、生间。"——《用间篇》

将所有调查结果综合为清晰的、可支撑决策的情报报告：
- 不是简单堆砌信息，而是提炼关键发现
- 明确标注"确认的事实"和"待验证的假设"
- 给出基于情报的建议方向
- 指明下一步需要验证的事项

## 常见错误

| 错误 | 孙子的警告 | 正确做法 |
|------|-----------|---------|
| 只知己不知彼（盲目自信） | "不知彼而知己，一胜一负" | 彼己并重，先知彼后知己 |
| 只知彼不知己（过度承诺） | 知彼知己缺一不可 | 诚实评估自身约束和能力 |
| 靠假设代替调查 | "不可象于事，不可验于度" | 靠实际调查获取情报 |
| 调查浅尝辄止 | 必须"知敌之情" | 深入到一手信息层面 |
| 调查完不形成结论 | 情报需服务于决策 | 综合为可执行的情报报告 |
| 忽视盲区存在 | "不知彼不知己，每战必殆" | 明确列出未知项和盲区 |

## 操作规程

当本 skill 被触发时，**必须输出"知彼知己表"**：

1. **声明调查对象**：一句话写出"此次情报调查的目标是：[具体问题/系统/对手]"
2. **输出双栏对比表**：

   ```
   知彼知己表：

   【知彼】—— 关于 [目标对象] 的发现：
   - 核心机制：……
   - 强项：……
   - 弱项：……
   - 近期变化/趋势：……
   - 关键约束：……

   【知己】—— 关于自身的评估：
   - 可用能力：……
   - 能力缺口：……
   - 可用资源：……
   - 硬约束：……
   - 历史教训：……

   【盲区】—— 仍然未知的关键信息：
   - [ ] [未知项1]——风险等级：[高/中/低]
   - [ ] [未知项2]——风险等级：[高/中/低]
   ```

3. **输出情报结论**，用固定格式收尾：
   ```
   情报结论：
   - 最佳攻击点（我强彼弱）：……
   - 最大风险点（我弱彼强）：……
   - 关键盲区及建议：……
   - 基于以上情报的行动建议：……
   ```

**知彼知己表必须先于任何方案设计出现，不得在信息不足时就给出方案。**

## 与其他 skill 的关系

- **庙算决策**（strategic-calculation）：庙算需要情报支撑，知彼知己为庙算提供五事七计的评估依据
- **虚实分析**（void-solid-analysis）：识别虚实的前提是知彼知己——不了解对手就无法判断虚实
- **因敌制胜**（adapt-to-win）：执行过程中环境变化，需要持续更新知彼知己的信息
- **奇正相生**（orthodox-unorthodox）：知己知彼后才能判断何时用正、何时用奇
