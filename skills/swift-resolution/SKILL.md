# 速战速决

> "故兵闻拙速，未睹巧之久也。夫兵久而国利者，未之有也。"
> —— 孙子《孙子兵法·作战篇》

## 核心原则

**用兵只听说过笨拙但迅速的，没有见过精巧却持久的。战争拖延而对国家有利的情况，从来没有过。速度胜过精巧，交付胜过完美。但"速"不等于"盲"——你必须先理解拖延的代价，才能正确选择速度。**

> "故兵贵胜，不贵久。"——《作战篇》
> "故不尽知用兵之害者，则不能尽知用兵之利也。"——《作战篇》

## 不适用场景

**以下情况不需要调用此 skill：**
- 质量不可妥协的场景（安全关键系统、数据一致性、核心金融逻辑）
- 正是因为之前"求快"才导致了当前问题——此时应该慢下来修正根因
- 基础性工作确实需要时间（架构设计、数据模型）——不要把必要的投入误判为拖延
- 已经在快速推进中，不存在拖延问题

## 何时使用

你应该在以下情况调用此 skill：

- 发现自己在反复推敲一个方案却迟迟不动手
- 陷入"再想想"的循环，分析瘫痪
- 追求完美方案，但 80% 的方案已经足够
- 需要快速修复一个问题（hotfix），而非完美重构
- MVP 交付——先上线再迭代
- 团队或项目的最大风险不是质量不够，而是交付太慢

## 方法流程

### 第一步：理解拖延的代价

> "故不尽知用兵之害者，则不能尽知用兵之利也。"——《作战篇》

在选择"快"之前，先量化"慢"的代价：
- 每拖延一天/一周，会消耗多少资源？
- 机会窗口是否在关闭？
- 团队士气是否因为迟迟看不到结果而下降？
- 拖延是否会引发新的问题或风险？

不了解"久战之害"，就不可能真正理解速度的价值。

### 第二步：判断——拙速还是巧久？

> "兵闻拙速，未睹巧之久也。"——《作战篇》

评估当前情况：
- 这是一个"拙速"胜过"巧久"的场景吗？
- 快速交付一个不完美的方案，后果可接受吗？
- 继续打磨带来的边际收益，是否值得拖延的成本？

**判断标准**：如果"不完美"的后果可修复、可迭代，则选择速度。

### 第三步：设定时限

> "兵贵胜，不贵久。"——《作战篇》

为当前任务设定明确的时间约束：
- 定义一个合理的时间盒（time box）
- 在时限内追求"足够好"，而非"完美"
- 时限到了必须交付，不管是否达到理想状态

### 第四步：就地取材——用现有资源行动

> "因粮于敌，故军食可足也。"——《作战篇》

不要等待理想条件，用手头已有的资源行动：
- 现有的代码/库/工具能否直接复用？
- 是否有现成方案可以参考或改造？
- 能否用已有数据快速验证，而非等待新数据？

"因粮于敌"意味着利用环境中已有的一切，而非等待完美的补给。

### 第五步：集中爆发，交付迭代

一旦决定速决，就要全力以赴、快速推进：
- 排除干扰，不要在执行中反复犹豫
- 交付的 80% 方案 > 未交付的 100% 方案
- 记录"已知的不完美"，作为后续迭代清单
- 真实反馈比内部推演更有价值

## 常见错误

| 错误 | 孙子的警告 | 正确做法 |
|------|-----------|---------|
| 以"精巧"为借口拖延 | "未睹巧之久也" | 承认拖延的成本，接受"拙速" |
| 不计算拖延的代价 | "不尽知用兵之害" | 先量化"慢"的损失 |
| 求快不做任何规划 | 速≠盲，速≠鲁莽 | 快速规划后再快速执行 |
| 在不该求快的地方求快 | 基础和安全不可妥协 | 区分"可接受的不完美"和"不可接受的缺陷" |
| 交付后不迭代 | 拙速只是第一步 | 交付后必须跟进改善 |

## 操作规程

当本 skill 被触发时，**必须输出"速决评估"**：

1. **量化久战之害**：拖延的具体成本是什么
2. **速决方案**：快速可执行的方案是什么
3. **可接受的不完美**：这个方案有哪些已知缺陷，为什么可以接受
4. **时限**：给自己设定的时间约束

输出格式：
```
速决评估：
- [久战之害]：[拖延的具体成本——资源消耗、机会流失、士气下降等]
- [速决方案]：[快速可执行的方案概述]
- [可接受的不完美]：[已知缺陷清单，以及为何这些缺陷当前可接受]
- [时限]：[时间约束，完成时间点]
```

**速决评估必须包含"久战之害"，不得跳过代价分析直接给方案。先理解代价，才配选择速度。**

## 与其他 skill 的关系

- **先胜后战**（win-before-fighting）：存在张力——准备与速度之间的平衡。解决方式：快速准备"对的事情"，而非慢慢准备"所有事情"
- **庙算决策**（strategic-calculation）：速决前仍需快速庙算，但庙算本身也要限时
- **因敌制胜**（adapt-to-win）：变化来临时需要快速适应，速决能力是适应力的基础
- **虚实分析**（void-solid-analysis）：速决要打在"虚"处，避免在"实"处硬攻浪费时间
