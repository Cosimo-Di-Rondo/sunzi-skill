# 庙算决策

> "夫未战而庙算胜者，得算多也；未战而庙算不胜者，得算少也。多算胜，少算不胜，而况于无算乎！"
> —— 孙子《孙子兵法·始计篇》

## 核心原则

**任何重大行动之前，必须在"庙堂"之上进行系统的计算与评估。胜负在开战之前就已经由算计的多寡决定了——算得多者胜，算得少者败，何况完全不算的人！**

孙子将战前评估归纳为"五事七计"：以道、天、地、将、法五个维度全面审视，再通过七项对比计算胜算。庙算不是犹豫，而是用理性取代冲动，用计算取代赌博。

## 不适用场景

**以下情况不需要调用此 skill：**
- 简单的执行任务——用户的意图和方案都已明确，直接做即可
- 用户已经完成了决策，只需要帮助实施
- 琐碎的、不涉及资源投入和风险的任务
- 紧急修复且问题和方案都很清晰——先止血，后复盘

## 何时使用

你应该在以下情况调用此 skill：

- 面临一个重大决策：做还是不做、选 A 还是选 B
- 项目启动前需要评估可行性
- 技术选型或架构方案需要系统比较
- 资源有限，需要判断这件事"值不值得做"
- 感觉自己在"无算"——凭直觉行动，缺乏系统评估
- 需要给出 go/no-go 建议

## 方法流程

### 第一步：明确目标——何为此"战"

> "兵者，国之大事，死生之地，存亡之道，不可不察也。"——《始计篇》

在计算之前，先清晰定义：
- 这次行动的目标是什么？要解决什么问题？
- 成功的标准是什么？失败的后果是什么？
- 这是一场"必须打的仗"还是"可以选择不打的仗"？

### 第二步：五事评估

> "一曰道，二曰天，三曰地，四曰将，五曰法。"——《始计篇》

逐项评估五个核心维度：
- **道**（使命对齐）：这件事与团队/组织的核心目标一致吗？上下能否同心？
- **天**（时机环境）：时机是否恰当？市场环境、技术趋势是否有利？
- **地**（地形条件）：基础设施、技术栈、代码现状是否支撑此行动？
- **将**（领导能力）：团队是否具备完成此事的能力和经验？
- **法**（组织流程）：制度、流程、工具链是否就绪？执行纪律是否可保证？

### 第三步：七计对比

> "主孰有道？将孰有能？天地孰得？法令孰行？兵众孰强？士卒孰练？赏罚孰明？吾以此知胜负矣。"——《始计篇》

将七项对比映射到当前场景，逐一比较我方与对手（或问题难度）的优劣：

| 七计 | 映射 | 我方评估 |
|------|------|---------|
| 主孰有道 | 使命/方向是否正确 | ? |
| 将孰有能 | 核心人员能力是否胜任 | ? |
| 天地孰得 | 时机和环境是否有利 | ? |
| 法令孰行 | 流程制度是否有效执行 | ? |
| 兵众孰强 | 资源（人力/算力/资金）是否充足 | ? |
| 士卒孰练 | 团队熟练程度和准备状态 | ? |
| 赏罚孰明 | 激励机制是否清晰合理 | ? |

### 第四步：计算胜算

> "多算胜，少算不胜，而况于无算乎！"——《始计篇》

综合五事七计的评估结果，量化判断：
- 有利因素有几项？不利因素有几项？
- 关键维度（道、将）是否占优？
- 整体胜算是"多算"还是"少算"？

### 第五步：做出决策——打还是不打

基于算计结果做出明确决策：
- **多算胜** → Go：制定行动计划，进入执行
- **少算不胜** → No-Go：说明原因，建议改善条件后再议
- **胜算不明** → 补充调查（调用 `know-both-sides`），获取更多信息后重新评估

### 第六步：若决定行动，设计方案

> "兵者，诡道也。故能而示之不能，用而示之不用，近而示之远，远而示之近。"——《始计篇》

决定行动后，方案设计应包含灵活性和预案：
- 主攻方向与备选方案
- 关键风险点与应对预案
- 阶段性检查点（何时重新评估）

## 常见错误

| 错误 | 孙子的警告 | 正确做法 |
|------|-----------|---------|
| 不评估就行动 | "无算"必败 | 先庙算，后行动 |
| 只看单一因素 | 五事缺一不可 | 五个维度全面评估 |
| 盲目乐观 | "少算不胜" | 诚实面对不利因素 |
| 忽视不利结论 | 庙算不胜则不应战 | 接受 No-Go 也是正确决策 |
| 计算过度不行动 | 兵贵神速，不宜久 | 算清即动，不要无限分析 |

## 操作规程

当本 skill 被触发时，**必须输出"庙算评估表"**：

1. **声明评估对象**：一句话写出"此次庙算评估的是：[具体决策/项目/方案]"
2. **五事评分**：对每个维度给出评分（优势/中性/劣势）并附一句话理由
   ```
   庙算评估表：
   - 道（使命对齐）：[优势/中性/劣势] — [理由]
   - 天（时机环境）：[优势/中性/劣势] — [理由]
   - 地（地形条件）：[优势/中性/劣势] — [理由]
   - 将（领导能力）：[优势/中性/劣势] — [理由]
   - 法（组织流程）：[优势/中性/劣势] — [理由]
   ```
3. **胜算判断**：综合评估后给出"多算/少算/不明"
4. **决策结论**，用固定格式收尾：
   ```
   庙算结论：[Go / No-Go / 需补充调查]
   关键理由：[一句话核心依据]
   若 Go，主攻方向：[行动要点]
   若 No-Go，改善建议：[需要提升的维度]
   ```

**庙算评估表必须先于任何行动方案出现，不得跳过评估直接给方案。**

## 与其他 skill 的关系

- **知彼知己**（know-both-sides）：庙算需要充分的情报支撑，信息不足时先调用知彼知己获取情报
- **虚实分析**（void-solid-analysis）：庙算决定"打不打"，虚实分析决定"打哪里"
- **先胜后战**（win-before-fighting）：庙算判定条件不足时，先创造有利条件再战
- **因敌制胜**（adapt-to-win）：庙算是战前计划，执行中需根据变化动态调整
